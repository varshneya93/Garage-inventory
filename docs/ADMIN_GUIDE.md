## Getting Started

### Accessing the Admin Dashboard

1. Navigate to `https://your-domain.com/admin`
2. Log in with your admin credentials
3. You'll be redirected to the admin dashboard

### First-Time Setup

1. **Update Your Profile**
   - Click on your profile icon
   - Update your name and email
   - Set a strong password

2. **Configure Theme**
   - Go to Settings → Theme
   - Customize colors and fonts
   - Preview changes before saving

3. **Set Up Integrations** (Optional)
   - Go to Settings → Integrations
   - Connect GitHub, social media, email services

## Managing Projects

### Creating a New Project

1. Click **Projects** in the sidebar
2. Click **Create New Project** button
3. Fill in project details:
   - **Title**: Project name (required)
   - **Description**: Short description for preview
   - **Content**: Detailed project information
   - **Status**: Draft or Published
   - **Featured**: Toggle to feature on homepage
   - **Tags**: Add relevant tags
   - **Categories**: Select project categories

4. Upload project images:
   - Click **Upload Images** or drag and drop
   - Add alt text for accessibility
   - Reorder images by dragging

5. Add project links:
   - Live demo URL
   - GitHub repository
   - Case study link

6. Click **Save as Draft** or **Publish**

### Editing Projects

1. Go to **Projects** page
2. Click on the project you want to edit
3. Make your changes
4. Click **Update** to save

### Publishing Projects

- **Draft**: Only visible to admins
- **Published**: Visible to all visitors
- **Archived**: Hidden but not deleted

To publish a draft:
1. Edit the project
2. Change status to **Published**
3. Set publish date (optional)
4. Click **Publish**

### Deleting Projects

1. Go to **Projects** page
2. Click the delete icon on the project
3. Confirm deletion
4. Note: This action cannot be undone

### Project Best Practices

- Use high-quality images (recommended: 1200x800px)
- Write clear, concise descriptions
- Add relevant tags for better searchability
- Include live demo and GitHub links when available
- Use featured projects sparingly (3-5 max)

## Managing Blog Posts

### Creating a Blog Post

1. Click **Blog** in the sidebar
2. Click **Create New Post** button
3. Fill in post details:
   - **Title**: Post title (required)
   - **Slug**: URL-friendly version (auto-generated)
   - **Excerpt**: Short summary for listings
   - **Content**: Full post content (Markdown supported)
   - **Status**: Draft or Published
   - **Featured**: Toggle to feature on blog homepage
   - **Tags**: Add relevant tags
   - **Categories**: Select post categories

4. Use the rich text editor:
   - Format text (bold, italic, headings)
   - Add images
   - Insert code blocks
   - Add links
   - Create lists

5. Set SEO metadata:
   - Meta description
   - Keywords
   - Social media preview image

6. Click **Save as Draft** or **Publish**

### Markdown Support

The blog editor supports Markdown:

```markdown
# Heading 1
## Heading 2

**Bold text**
*Italic text*

- List item 1
- List item 2

[Link text](https://example.com)

![Image alt text](image-url.jpg)

`inline code`

```code block```
```

### Scheduling Posts

1. Edit your blog post
2. Set status to **Published**
3. Set **Publish Date** to future date
4. Click **Schedule**
5. Post will automatically publish at scheduled time

### Blog Best Practices

- Write engaging titles (50-60 characters)
- Use clear headings for structure
- Add images to break up text
- Write meta descriptions (150-160 characters)
- Use tags consistently
- Proofread before publishing
- Update old posts to keep content fresh

## Media Management

### Uploading Media

1. Click **Media** in the sidebar
2. Click **Upload** or drag and drop files
3. Supported formats: JPG, PNG, WebP, GIF
4. Maximum file size: 10MB per file

### Organizing Media

- **Search**: Find media by filename
- **Filter**: By type, date, or usage
- **Sort**: By date, name, or size
- **Tags**: Add tags for organization

### Using Media

- Click on any media to copy URL
- Insert into projects or blog posts
- View usage to see where media is used

### Media Best Practices

- Use descriptive filenames
- Optimize images before uploading
- Add alt text for accessibility
- Delete unused media regularly
- Use WebP format for better performance

## Contact Inquiries

### Viewing Inquiries

1. Click **Inquiries** in the sidebar
2. View all contact form submissions
3. Filter by status:
   - **New**: Unread inquiries
   - **Read**: Viewed but not responded
   - **Responded**: Reply sent
   - **Closed**: Resolved inquiries

### Managing Inquiries

1. Click on an inquiry to view details
2. View sender information and message
3. Add internal notes
4. Change status
5. Mark as spam if needed

### Responding to Inquiries

1. Open the inquiry
2. Click **Reply** button
3. Compose your response
4. Click **Send**
5. Inquiry status automatically updates to **Responded**

### Inquiry Best Practices

- Respond within 24-48 hours
- Use professional, friendly tone
- Add notes for future reference
- Close inquiries when resolved
- Review spam regularly

## Troubleshooting

### Can't Upload Images

- Check file size (max 10MB)
- Verify file format (JPG, PNG, WebP, GIF)
- Clear browser cache
- Try different browser

### Changes Not Appearing

- Clear browser cache
- Hard refresh (Ctrl/Cmd + Shift + R)
- Check if changes were saved
- Verify publish status

### Login Issues

- Verify credentials
- Reset password if needed
- Clear cookies
- Contact administrator
